# DriverSearch

## Общая информация о проекте

 Проект  выполнен на языке программирования **C#** в среде **Visual Studio**.  
Цель проекта — реализовать механизм подбора ближайших водителей к заказу на карте, а также сравнить различные алгоритмы поиска по производительности и корректности работы.

Проект разработан в рамках задания по изучению:
- алгоритмов
- работы с коллекциями
- модульного тестирования
- измерения производительности кода
##  Условие задачи

По условию задания необходимо реализовать механизм подбора водителя на заказ.

Карта представляет собой прямоугольную сетку размером **N × M**, состоящую из квадратных ячеек размером **1 × 1**.  
В каждой ячейке может находиться **только один водитель**.

Каждый водитель характеризуется:
- уникальным идентификатором `Id`
- координатами `X` и `Y`

Координаты водителей и заказа задаются целыми числами, где:
- `0 ≤ X < N`
- `0 ≤ Y < M`

Задача алгоритма — по координатам заказа найти **5 ближайших водителей**.

##  Описание модели данных

### Класс Driver

Класс `Driver` представляет собой модель водителя.

Содержит следующие свойства:
- `Id` — уникальный идентификатор водителя
- `X` — координата водителя по оси X
- `Y` — координата водителя по оси Y

Все свойства доступны только для чтения, что предотвращает случайное изменение данных.

## Вычисление расстояния

Для определения близости водителя к заказу используется **евклидово расстояние**.

Формула расчёта:

distance = √((x1 - x2)² + (y1 - y2)²)

##  Реализованные алгоритмы поиска

###  Алгоритм 1: SortSearch

Алгоритм `SortSearch` работает следующим образом:
1. Создаётся копия списка водителей
2. Весь список сортируется по расстоянию до точки заказа
3. Из отсортированного списка выбираются первые 5 водителей

**Преимущества:**
- простота реализации
- понятная логика

**Недостатки:**
- сортируется весь список, что может быть неэффективно при большом количестве данных

###  Алгоритм 2: ManualSearch

Алгоритм `ManualSearch` реализует поиск вручную:
1. Из списка выбирается ближайший водитель
2. Найденный водитель добавляется в результат
3. Водитель удаляется из временного списка
4. Процесс повторяется 5 раз

**Преимущества:**
- не требуется сортировка всего списка
- наглядная логика поиска

**Недостатки:**
- более сложный код
- больше операций при большом количестве данных

###  Алгоритм 3: QueueSearch

Алгоритм `QueueSearch` использует возможности LINQ:
1. Список сортируется с помощью `OrderBy`
2. Выбираются первые 5 элементов с помощью `Take(5)`

**Преимущества:**
- компактный и читаемый код
- использование стандартных средств .NET

**Недостатки:**
- использует сортировку списка

##  Модульное тестирование

Для тестирования используется библиотека **NUnit**.

Были реализованы тесты, которые проверяют:
- что каждый алгоритм возвращает ровно 5 водителей
- что ближайший водитель определяется корректно
- что методы корректно обрабатывают входные данные

Все алгоритмы поиска полностью покрыты тестами.

##  Сравнение производительности

Для сравнения скорости работы алгоритмов используется библиотека **BenchmarkDotNet**.

В рамках бенчмарков:
- создаётся список из 50 000 водителей
- выполняется поиск ближайших водителей
- замеряется время выполнения каждого алгоритма
- анализируется потребление памяти
