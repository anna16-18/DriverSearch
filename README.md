# DriverSearch

## 1. Введение

Данный репозиторий содержит проект, выполненный на языке программирования C# в среде разработки Visual Studio.  
Проект реализует механизм подбора ближайших водителей к заказу на карте в соответствии с заданными условиями.

Работа выполнена  с целью:
- изучения различных алгоритмов поиска
- сравнения их производительности
- освоения модульного тестирования
- работы с системой контроля версий Git и сервисом GitHub

## 2. Условие задачи 

По условию задания необходимо реализовать механизм подбора водителя на заказ.

### 2.1 Описание карты

Карта представляет собой прямоугольную сетку размером N × M.  
Сетка состоит из квадратных ячеек размером 1 × 1.

В каждой ячейке:
- может находиться не более одного водителя
- координаты ячейки задаются числами X и Y

Допустимые значения координат:
- `0 ≤ X < N`
- `0 ≤ Y < M`

### 2.2 Описание водителя

Каждый водитель имеет:
- уникальный идентификатор `Id`
- координату `X`
- координату `Y`

Идентификатор водителя уникален и используется для его однозначного определения.

### 2.3 Описание заказа

Заказ задаётся:
- координатой `X`
- координатой `Y`

По этим координатам необходимо определить пять ближайших водителей.

## 3. Цели и задачи проекта

В рамках выполнения проекта были поставлены следующие задачи:

1. Реализовать три различных алгоритма поиска ближайших водителей  
2. Реализовать единый механизм вычисления расстояния  
3. Сравнить алгоритмы по производительности  
4. Покрыть каждый алгоритм автоматическими тестами  
5. Оформить результаты в репозитории GitHub  

## 4. Структура решения

Решение `DriverSearch` состоит из трёх отдельных проектов.

### 4.1 Проект DriverSearch (основной проект)

Основной проект содержит реализацию логики приложения и алгоритмов поиска.

#### Содержимое проекта:

- `Driver.cs`  
  Класс, описывающий водителя. Содержит уникальный идентификатор и координаты.

- `Distance.cs`  
  Статический класс для вычисления расстояния между двумя точками на карте.

- `SortSearch.cs`  
  Алгоритм поиска ближайших водителей с использованием сортировки списка.

- `ManualSearch.cs`  
  Алгоритм ручного поиска ближайших водителей без полной сортировки списка.

- `QueueSearch.cs`  
  Алгоритм поиска с использованием LINQ (`OrderBy` и `Take`).

- `Program.cs`  
  Консольное приложение для демонстрации работы всех алгоритмов.

### 4.2 Проект DriverSearch.Benchmark

Проект предназначен для сравнения производительности реализованных алгоритмов.

Для измерений используется библиотека BenchmarkDotNet.

В процессе бенчмарка:
- создаётся список из 50 000 водителей
- для каждого алгоритма выполняется поиск
- измеряется время выполнения и использование памяти


### 4.3 Проект DriverSearch.Tests

Проект содержит модульные тесты, написанные с использованием библиотеки NUnit.

Назначение тестов:
- проверка корректности работы алгоритмов
- автоматическая проверка логики при изменении кода


## 5. Описание модели данных

### 5.1 Класс Driver

Класс `Driver` представляет модель водителя.

Содержит следующие свойства:
- `Id` — уникальный идентификатор
- `X` — координата по оси X
- `Y` — координата по оси Y

Все свойства доступны только для чтения, что исключает случайное изменение данных.

## 6. Вычисление расстояния между точками

Для определения близости водителя к заказу используется евклидово расстояние.

Формула:

distance = √((x1 - x2)² + (y1 - y2)²)

Данная формула реализована в классе `Distance` и используется во всех алгоритмах поиска.

## 7. Реализованные алгоритмы поиска

### 7.1 Алгоритм SortSearch

Алгоритм работает следующим образом:
1. Создаётся копия списка водителей
2. Весь список сортируется по расстоянию до точки заказа
3. Возвращаются первые 5 водителей

Преимущества:
- простота реализации
- понятная логика

Недостатки:
- сортируется весь список, что снижает производительность при больших объёмах данных

### 7.2 Алгоритм ManualSearch

Алгоритм реализует ручной поиск ближайших водителей:
1. Находится ближайший водитель
2. Он добавляется в результат
3. Удаляется из временного списка
4. Процесс повторяется 5 раз

Преимущества:
- не требуется сортировка всего списка
- более наглядная реализация алгоритма

Недостатки:
- более сложная реализация
- увеличенное количество операций

### 7.3 Алгоритм QueueSearch

Алгоритм использует возможности LINQ:
1. Список сортируется с помощью `OrderBy`
2. Выбираются первые 5 элементов с помощью `Take`

Преимущества:
- компактный и читаемый код
- использование стандартных средств .NET

**Недостатки:**
- выполняется сортировка списка


## 8. Тестирование

Тесты проверяют:
- что каждый алгоритм возвращает ровно 5 водителей
- что ближайший водитель определяется корректно
- корректную работу методов поиска

Все реализованные алгоритмы полностью покрыты тестами.
